# Contributing to StateSight

Thanks for contributing.

This repository is currently in a foundation stage, so we are optimizing for clarity and good collaboration habits.

## Ground Rules

- `main` is protected. Do not push directly to `main`.
- Create a branch for every change.
- Keep pull requests small and focused.
- Ask early if a change may affect structure, naming, or workflow.

## Branch Naming

Use short, clear branch names:

- `docs/<topic>`
- `chore/<topic>`
- `feat/<topic>`
- `fix/<topic>`

Example: `docs/update-roadmap`

## Commit Expectations

- Write clear commit messages.
- Keep each commit focused on one logical change.
- Avoid mixing unrelated changes in one commit.

## Pull Request Expectations

1. Explain what changed.
2. Explain why it changed.
3. Note what you tested (or if not tested yet).
4. Ask for feedback where you are unsure.

Use the PR template in `.github/pull_request_template.md`.

## New Contributors

Beginners are welcome. If anything is unclear, ask in the PR or issue before spending too much time stuck.
